# 🍼 もぐもぐダイアリー

離乳食の献立管理・冷凍ストック作成スケジュール・買い物リスト・アレルギー記録を一括管理できるWebアプリです。

夫婦2人のスマホからリアルタイムで同期できます。

---

## ✨ 機能

### 📅 献立タブ
- カレンダーから日付を選んで献立を入力
- 食材ごとに量（小さじ / g）を設定
- 初めての食材に「初」マーク
- 「＋まぜる」で混ぜて食べさせる食材を表示
- 食べた量（少なめ / ふつう / よく食べた）と反応メモを記録
- 1回食 / 2回食 / 3回食に対応

### 🍳 作るものタブ
- 献立から自動で冷凍ストック作成スケジュールを計算
- **前日に準備** → 翌日〜7日以内の同じ食材をまとめて作成
- 冷凍ストック（🧊）と当日調理（🔥）を自動分類
- 各食材の合計量・内訳（小1×2、小3×1 等）・使用日を表示
- 1週間単位で表示

### 🛒 買い物リスト
- 献立から必要な食材を自動リストアップ
- カテゴリ別に整理（穀類・野菜・果物・タンパク質・その他）

### 📋 アレルギー記録
- 全食材のアレルギーチェック状態を管理
- OK（✅）/ 様子見（👀）/ NG（🚫）/ 未確認（❓）
- NG食材は献立入力時に選択不可（自動ブロック）

### ⚙️ 設定
- 誕生日入力 → 生後月数・離乳食ステージを自動表示
- 1回食〜3回食の切替
- 食材の追加（カスタム食材）
- 冷凍しない食材の管理（豆腐・バナナ等）

## 🚀 セットアップ手順

### 1. Firebase プロジェクトの作成

1. [Firebase Console](https://console.firebase.google.com/) にアクセス
2. 「プロジェクトを追加」→ 好きな名前を入力 → Google Analytics はオフでOK
3. 左メニュー「構築」→「Realtime Database」→「データベースを作成」
   - ロケーション: `asia-southeast1`（シンガポール）推奨
   - セキュリティルール: **テストモードで開始**
4. 「ルール」タブで以下に変更して「公開」:

```json
{
  "rules": {
    ".read": true,
    ".write": true
  }
}
```

5. プロジェクト概要 →「</>」（Webアプリ追加）→ アプリ名を入力 → 「アプリを登録」
6. 表示される `firebaseConfig` の値をメモ

### 2. index.html の設定

`index.html` を開いて、先頭付近の `firebaseConfig` を自分のプロジェクトの値に置き換え:

```js
const firebaseConfig = {
  apiKey: "あなたのAPIキー",
  authDomain: "あなたのプロジェクト.firebaseapp.com",
  databaseURL: "https://あなたのプロジェクト-default-rtdb.xxx.firebasedatabase.app",
  projectId: "あなたのプロジェクト",
  storageBucket: "あなたのプロジェクト.firebasestorage.app",
  messagingSenderId: "あなたの番号",
  appId: "あなたのAppID"
};
```

> ⚠️ `databaseURL` が `firebaseConfig` に含まれていない場合は、Realtime Database のページから URL をコピーして追加してください。

### 3. GitHub Pages にデプロイ

1. [GitHub](https://github.com/new) でリポジトリを作成（「Add a README file」にチェック）
2. 「Add file」→「Upload files」で `index.html` をアップロード
3. 「Settings」→「Pages」→ Branch: `main` / `/ (root)` → 「Save」
4. 数分後に `https://あなたのユーザー名.github.io/リポジトリ名/` でアクセス可能

### 4. スマホで使う

- URLをLINE等で共有
- iPhone: Safari → 共有ボタン（□↑）→「ホーム画面に追加」でアプリのように使えます

## 👨‍👩‍👧 家族IDについて

初回アクセス時に**家族ID**（6桁のランダム英数字）が発行されます。

- **パートナーと共有**: 同じURLを開いて「家族IDで参加」から入力すると、2人のデータがリアルタイム同期されます
- **他の家族も利用可能**: 別の家族が同じURLにアクセスしても、家族IDが異なるためデータは完全に分離されます
- 設定タブで家族IDの確認・変更ができます

### 友人に教える場合

1. アプリのURL（`https://あなたのユーザー名.github.io/リポジトリ名/`）を教える
2. 友人が初回アクセスすると「新しい家族を作成」画面が表示される
3. 友人が自分の家族IDを作成すれば、あなたのデータとは独立して使えます

> 💡 Firebase の同じプロジェクトを共有していても、家族IDごとにデータが分離されるため安心です。
